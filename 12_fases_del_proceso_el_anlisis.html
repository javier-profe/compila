1.2.- Fases del proceso: el análisis

Antes de generar el archivo final, el compilador divide su trabajo en fases. La primera es el análisis, donde se comprueba que el código esté bien escrito, no falten puntos y coma, y las variables existan.

El compilador debe realizar 3 tipos de análisis (ALX, AST y ASM) antes de pasar a la siguiente fase de síntesis donde, de no existir errores, obtenemos el código máquina u objeto.

Tipos de análisis
Análisis Características
Lexicográfico Comprueba que el código fuente está escrito correctamente.
Sintáctico Verifica que los elementos obtenidos en el ALX se relacionan respetando la sintaxis del lenguaje de programación utilizado.
Semántico Asegura que el código fuente, además de estar bien escrito y respetar las reglas de sintaxis del lenguaje usado, tiene sentido. 

¡Importante!

El análisis léxicográfico y sintáctico detecta errores de escritura, pero no asegura que el programa haga lo que tú quieres (lógica).

El análisis semántico detecta situaciones de inconsistencia tal como sucede con este texto: Los elefantes bebían neumáticos. Todo, aparentemente, es correcto, tanto lexicográfica como sintácticamente, pero nos chirría porque falla la semántica (los neumáticos no tienen el atributo o cualidad de ser bebibles).
